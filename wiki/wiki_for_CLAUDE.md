Есть сайт https://svremya.su/

Данная wiki реализована на стандартном API https://m.mediawiki.org/wiki/API:REST_API/ru

Необходимо в модуле wiki реализовать следующую логику:

[RootWiki.kt](src/main/java/su/sv/wiki/root/RootWiki.kt)
В RootWiki сделать view поиска текста. При вводе в поле поиска текст необходимо показывать загрузку, а после
отображать результат, если он есть в БЛОКЕ ПРОСМОТРА. Если результата нет, выводить текст "Ничего не найдено".
В качестве результата должна быть статья или слово с сайта с её развёрнутым описанием

БЛОК ПРОСМОТРА отображается только когда было введено слово в поле поиска

Если БЛОКА ПРОСМОТРА нет, то отображать список из последних успешных запросов, которые были ранее сохранены в локальное хранилище.


Например, при вводе слова "Конечное" со страницы, которую выдаёт поиск (https://svremya.su/%D0%9A%D0%BE%D0%BD%D0%B5%D1%87%D0%BD%D0%BE%D0%B5)
должен быть выведен текст: 

"Конечное - это нечто, взятое со своей имманентной границей. (Имманентная - внутренне присущая.)

Моменты конечного - долженствование и предел."

При этом, как и на сайте, при клике на слова с гиперссылкой (например, "нечто" или "предел") должен обновляться БЛОК ПРОСМОТРА с 
отображением результата поиска по этим словам.


Для найденного результата в БЛОКЕ ПРОСМОТРА должна быть кнопка добавления в избранное. При её нажатии данные сохраняются 
в локальное хранилище и более будут доступны автономно даже без интернета

Условия:

1. Весь UI делать на compose
2. Необходимо реализовать логика парсинга данных с сайта в текстовый вид, чтобы можно было отобразить 
результаты поиска и сам поиск БЕЗ WebView
3. Задачу необходимо максимально декомпозировать и выполнять поэтапно. После каждого этапа необходимо требовать проверку от меня
4. Локальное хранилище использовать SharedPreferences или Room
