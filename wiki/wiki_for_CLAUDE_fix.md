# Формализованная задача: Wiki-модуль для приложения SV APP

## 1. Общая информация

### 1.1 Источник данных
- **Сайт**: https://svremya.su/
- **API**: MediaWiki Classic API (`/api.php`)
- **Тип контента**: Статьи и определения (словарь марксизма, диалектика)

### 1.2 Используемые API-эндпоинты

#### Поиск статей
```
GET https://svremya.su/api.php?action=query&list=search&srsearch={query}&format=json&utf8=
```
**Ответ**:
```json
{
  "query": {
    "search": [
      {
        "title": "Конечное",
        "pageid": 223,
        "snippet": "Конечное - это [[нечто]], взятое со своей..."
      }
    ]
  }
}
```

#### Получение содержимого страницы (HTML с ссылками)
```
GET https://svremya.su/api.php?action=parse&page={title}&prop=text|links&format=json
```
**Ответ**:
```json
{
  "parse": {
    "title": "Конечное",
    "pageid": 223,
    "text": {
      "*": "<div class=\"mw-parser-output\"><p>Конечное - это <a href=\"/Нечто\">нечто</a>..."
    },
    "links": [
      {"*": "Граница"},
      {"*": "Нечто"}
    ]
  }
}
```

#### Автодополнение поиска (OpenSearch)
```
GET https://svremya.su/api.php?action=opensearch&search={query}&limit=5&format=json&utf8=
```
**Ответ** (JSON-массив):
```json
[
  "Маркс",
  ["Маркс, Карл", "Марксизм"],
  ["", ""],
  ["https://svremya.su/Маркс,_Карл", "https://svremya.su/Марксизм"]
]
```
- `[0]` - поисковый запрос
- `[1]` - массив заголовков статей (подсказки)
- `[2]` - массив описаний (обычно пустые)
- `[3]` - массив URL

## 2. Функциональные требования

### 2.1 Экран RootWiki

#### Компоненты UI:
1. **Поле поиска** (SearchBar)
   - Располагается в верхней части экрана
   - **Автоматический поиск**: при вводе текста от 3 символов автоматически отправлять запрос к API
   - **Debounce**: запрос отправляется через 1.5 секунды после прекращения ввода
   - **Автоподсказки**: при вводе текста от 2 символов показывать подсказки под полем поиска
   - При поиске показывать индикатор загрузки
   - После загрузки отображать результат в БЛОКЕ ПРОСМОТРА
   - Кнопка очистки (крестик) для быстрого удаления текста

2. **Подсказки поиска** (SearchSuggestions)
   - Отображаются под полем поиска при вводе текста (от 2 символов)
   - Максимум 5 подсказок
   - **Анимация появления**: плавное увеличение высоты сверху вниз (300мс)
   - **Анимация исчезновения**: плавное уменьшение высоты (200мс)
   - При клике на подсказку → загрузка статьи
   - Источник данных: MediaWiki OpenSearch API

3. **БЛОК ПРОСМОТРА** (ArticleView)
   - Отображается только когда был введён текст в поиск
   - Содержит:
     - Заголовок статьи
     - Текст статьи с кликабельными гиперссылками
     - Кнопку "Добавить в избранное"
   - При клике на гиперссылку → загрузка новой статьи в этот же блок
   - **Вертикальный скролл** для длинного текста
   - **Выделение и копирование текста** (SelectionContainer)
   - **Внешние ссылки** (видео, источники) выделены другим цветом и открываются в браузере

4. **Список истории** (HistoryList)
   - Отображается когда БЛОК ПРОСМОТРА пуст (нет активного поиска)
   - Показывает последние успешные запросы
   - При клике на элемент → загрузка статьи в БЛОК ПРОСМОТРА

5. **Иконка избранного**
   - Отображается справа от поля поиска
   - Видна только при наличии избранных статей
   - При нажатии → переход на экран "Избранное"
   - Плавная анимация появления/исчезновения

#### Состояния экрана:
```
sealed class UiWikiState {
    object Initial : UiWikiState()           // Начальное состояние, показываем историю
    object Loading : UiWikiState()           // Загрузка
    data class Content(                      // Статья найдена
        val article: UiWikiArticle,
        val isFavorite: Boolean
    ) : UiWikiState()
    object NotFound : UiWikiState()          // Ничего не найдено
    data class Error(val message: String)    // Ошибка
}
```

### 2.2 Сценарии использования

#### Сценарий 1: Автозаполнение поиска (подсказки)
1. Пользователь начинает вводить текст в поле поиска
2. После ввода 2-го символа система запрашивает подсказки через OpenSearch API
3. Под полем поиска плавно появляется список подсказок (максимум 5)
4. Пользователь может выбрать подсказку или продолжить ввод
5. При выборе подсказки → загрузка статьи

#### Сценарий 2: Автоматический поиск статьи (debounce)
1. Пользователь начинает вводить текст в поле поиска
2. После ввода 3-го символа система ожидает 1.5 секунды
3. Если пользователь не продолжает ввод — отправляется запрос к API
4. Показывается индикатор загрузки
5. При успехе → отображается статья в БЛОКЕ ПРОСМОТРА
6. При отсутствии результата → текст "Ничего не найдено"
7. **При нажатии на крестик (очистка поля) → возврат к начальному состоянию (история поиска)**

#### Сценарий 3: Навигация по ссылкам
1. Пользователь кликает на гиперссылку в тексте статьи
2. Показывается индикатор загрузки
3. Загружается новая статья
4. БЛОК ПРОСМОТРА обновляется

#### Сценарий 4: Добавление в избранное
1. Пользователь нажимает кнопку "Добавить в избранное"
2. Статья сохраняется в локальную БД
3. Становится доступна оффлайн

#### Сценарий 5: Просмотр истории
1. При отсутствии активного поиска показывается список истории
2. История содержит последние успешные запросы
3. При клике на элемент истории → загрузка статьи

#### Сценарий 6: Переход по внешним ссылкам
1. В тексте статьи пользователь видит внешнюю ссылку (видео, источник)
2. Ссылка выделена другим цветом (tertiary)
3. При клике → открывается системный браузер/приложение

   - Заголовок: "Избранное"
   - Стрелка назад: возврат на RootWiki
   - Иконка корзины: очистка избранного (только при наличии элементов)

   - Элементы: названия статей с иконкой сердечка
   - При клике → переход на экран "Статья"

4. **Пустое состояние**
   - Иконка сердечка
   - Текст: "Избранное пусто"

```kotlin
sealed class FavoritesState {
```

## 5. Дополнительные требования

### 5.1 Обработка ошибок
- Нет интернета → показать ошибку с возможностью повтора
- Статья не найдена → текст "Ничего не найдено"
- Ошибка сервера → показать сообщение об ошибке

### 5.2 UX
- Debounce для поиска (не отправлять запрос на каждый символ)
- Показывать прогресс-индикатор при загрузке
- Анимации переходов между состояниями
- Pull-to-refresh для обновления статьи

### 5.3 Оффлайн-режим
- Избранные статьи доступны без интернета
- При отсутствии интернета показать только избранное и историю
