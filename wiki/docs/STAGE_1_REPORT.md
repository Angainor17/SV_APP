# Этап 1: Завершён

## Выполненные задачи

### 1. Исследованы API-эндпоинты

| Эндпоинт | Назначение | Статус |
|----------|------------|--------|
| `action=query&list=search` | Поиск статей | ✅ Протестирован |
| `action=parse` | Получение содержимого страницы | ✅ Протестирован |
| `action=opensearch` | Автодополнение | ✅ Протестирован |

### 2. Определены форматы ответов

- **Успешный поиск**: `query.search[]` с `title`, `pageid`, `snippet`
- **Не найдено**: `query.searchinfo.totalhits = 0`, `query.search = []`
- **Страница**: `parse.text["*"]` - HTML с ссылками, `parse.links[]` - список ссылок
- **Ошибка**: `error.code = "missingtitle"`, `error.info`

## Ключевые решения

1. **Поиск**: Использовать `srwhat=nearmatch` для точного совпадения по заголовку
2. **Получение контента**: Использовать `action=parse` вместо `action=query` (готовый HTML с ссылками)
3. **Обработка ссылок**: Парсить HTML-контент с регексом или использовать Jsoup
