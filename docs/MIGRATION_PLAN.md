# План миграции экранов к дизайн-системе

## Обзор

Этот документ описывает план поэтапного приведения всех экранов приложения к единой дизайн-системе, описанной в `DESIGN_SYSTEM.md`.

---

## Принципы миграции

1. **Постепенная миграция** — экраны обновляются последовательно, без breaking changes
2. **Приоритет по частоте использования** — сначала обновляются наиболее используемые экраны
3. **Совместимость** — старый API поддерживается через `@Deprecated` аннотации
4. **Тестирование** — каждый обновлённый экран проверяется в обеих темах

## Чек-лист для миграции каждого экрана

При миграции каждого экрана проверять:

- [ ] Импорты обновлены на новые компоненты
- [ ] Цвета берутся из `MaterialTheme.colorScheme`
- [ ] Отступы берутся из `LocalAppDimensions.current`
- [ ] Текстовые стили из `MaterialTheme.typography`
- [ ] Обработаны все состояния (Loading, Error, Content, Empty)

- [ ] Экран корректно отображается в светлой теме
